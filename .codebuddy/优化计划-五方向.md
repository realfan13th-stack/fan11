# 教师作息工作台 — 优化计划

> 存放日期：2026-06-27
> 项目文件：schedule_v103.html
> 状态：待选择推进方向

---

## 产品定位

教师个人作息工作台——围绕"教师日常高频使用的教学管理面板"这一核心场景，从视觉品质、实用功能、操作效率、移动便携四个维度全面升级。

---

## 方向一：UI/UX 设计重塑

- 视觉风格革新：从现有功能化 UI 升级为有教师职业气质的设计语言（温润、聚焦、秩序感）
- 色彩与字体系统重建：基于教师职业属性（黑板绿、粉笔白、纸张暖黄）提取专属设计令牌
- 左侧 Bento 面板精细化：时钟、状态、倒计时、作息四卡片的视觉层次和空间节奏优化
- 微交互与动效：为切换夏令/冬令、Tab 切换、作息折叠等高频操作添加细腻过渡动画
- 主题体系扩展：保留现有 5 套主题，新增"黑板"和"暖纸"两套专属主题

## 方向二：实用功能新增

- **代课管理**：记录代课日期、原教师、代课教师、科目、节次；自动同步到课程表标注
- **换课管理**：支持两两换课或单向调课，关联课表联动更新，支持撤销
- **周视图**：在现有日视图基础上，新增一周作息预览卡片，快速查看本周安排
- **任务/备忘模块**：在左侧面板新增简易待办卡片，贴近期末考试、公开课等教学节点
- **考试日程管理**：新增考试周特殊安排，关联作息表和倒计时

## 方向三：交互逻辑优化

- 拖拽排序：作息时段和条目支持拖拽重排，替代现有逐行表格编辑
- 快捷操作面板：在课表区新增右键菜单或悬浮按钮，快速添加代课/换课记录
- 一键切换：课表视图与晚托视图之间添加互跳快捷入口
- 撤销/重做：数据编辑操作支持全局撤销栈
- 今日焦点：页面加载时自动滚动到当前时段，高亮当前课节

## 方向四：移动端体验升级

- PWA 改造：添加 manifest.json 和 Service Worker，支持"添加到主屏幕"
- 离线可用：Service Worker 缓存全量资源，无网络也能正常使用
- 移动端原生操作感：底部导航栏、滑动手势、触觉反馈适配
- 主屏幕图标：设计教师工作台专属 App 图标
- 推送通知：利用 Notification API 实现上课提醒、作息变更通知

## 方向五：架构与运维优化

- 代码模块化：将单文件拆分为 HTML/CSS/JS 三文件（或使用 ES Modules）
- 自动备份：利用 IndexedDB 实现定时自动备份，替代纯手动 Excel 导出
- 云端同步：可选接入 localStorage 同步服务（如简单的 JSON 导入导出或 GitHub Gist）
- 性能优化：虚拟滚动晚托日历、懒加载节假日数据、减少 DOM 重排
- 版本管理：内置版本号显示和手动回退机制

---

## 技术栈现状

- **前端**：纯 HTML + CSS + JS 单文件，无框架，无构建工具
- **外部依赖（CDN）**：XLSX（Excel 读写）、html2canvas（图片导出）
- **存储**：localStorage
- **API**：timor.tech（节假日）

## 各方向技术方案概述

### 方向一（UI 重塑）
- 基于 CSS 变量系统的设计令牌
- 引入 CSS `@layer` 管理样式优先级
- 动效使用 CSS `transition` + `@keyframes`，遵守 `prefers-reduced-motion`
- 无需新增外部依赖

### 方向二（功能新增）
- 代课/换课数据模型在现有 `scheduleData` 下新增 `substitutions` 数组
- 与现有 category/type 体系对齐，复用渲染管线
- 周视图复用现有作息渲染逻辑，增加 `range` 参数

### 方向三（交互优化）
- 拖拽排序使用原生 Drag and Drop API
- 撤销栈使用轻量 Command 模式
- 快捷键使用 `KeyboardEvent` 绑定

### 方向四（PWA）
- 单文件应用场景下，采用内联 Service Worker 方案
- 使用 Blob URL 注册 Service Worker
- manifest.json 通过 `<link>` 内联 data URI
- 移动端适配复用现有响应式断点，重点优化 375px 宽度

### 方向五（架构）
- 拆分策略：保持 schedule_v103.html 为入口，通过 `<link>` + `<script src>` 引入外部 CSS/JS
- IndexedDB 封装为 StorageService 单例
- 版本管理使用 VERSION 常量 + loadData() 迁移链
