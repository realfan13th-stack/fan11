## 需求概述

对晚托表 UI 进行四项优化：学期时间模块去卡片化改为简洁行内文本风格，教师统计图例内容放大，导出图底部统计模块放大，导出弹窗按钮改为磨砂玻璃圆角胶囊风格。

## 核心功能

### 1. 学期时间模块简洁化

- 去除当前 `.semester-bar-card` 的卡片边框和背景
- 日期标签改为轻量行内文本展示，保留 `→` 箭头分隔
- 编辑学期按钮保留但样式融入整体风格（小号次按钮风格）
- 整体视觉从"卡片块"降级为"信息行"

### 2. 教师统计图例内容放大

- `.as-legend-item` 字号上限从 15px 提高到 16px
- `.as-legend-days` 字号上限从 12px 提高到 13px
- `.as-legend-days-total` 字号上限从 12px 提高到 13px
- 色块 `.as-legend-color` 尺寸上限从 14px 提高到 15px

### 3. 导出图底部统计模块放大

- 教师统计项字号从 12px 提高到 14px，padding 从 5px 9px 提高到 7px 12px
- 总计项字号从 12px 提高到 14px，padding 同步提高
- 统计容器 gap 从 10px 提高到 14px，padding 从 14px 18px 提高到 18px 22px

### 4. 导出弹窗按钮改为磨砂玻璃胶囊风格

- 主按钮：半透明 accent 色背景 + backdrop-filter 模糊 + 柔和边框 + 圆角胶囊 + hover 亮起
- 次按钮/取消按钮：半透明白色背景 + backdrop-filter 模糊 + 柔和边框 + 圆角胶囊 + 与主按钮等高等宽
- 取消按钮从极小的 text-only 按钮改为与主按钮一致的胶囊风格按钮

## 技术方案

### 实现策略

全部修改集中在单个文件 `schedule_v103.html` 内完成，无需新建文件或引入外部依赖。修改涉及 CSS 样式替换、JS 渲染逻辑调整、HTML 结构微调三类变更，按模块分组执行。

### 关键设计决定

- **学期栏去卡片化**：直接修改 CSS 移除 `border`/`background`/`border-radius`，改为 `padding: 0; background: transparent;` 行内文本风格，日期标签同理去边框和背景
- **磨砂玻璃按钮**：使用 CSS `backdrop-filter: blur()` 实现，无需 JS。主按钮使用 `rgba(89,122,181,0.12)` 半透明底色，次按钮使用 `rgba(255,255,255,0.6)`
- **字号放大**：直接修改 `clamp()` 的上限值，保持响应式兼容
- **导出图统计**：直接修改 JS 内联样式字符串中的数值

### 实现细节

#### 学期栏 CSS 修改（第362-388行）

- `.semester-bar-card`：去除 `background`、`border`、`border-radius`、`padding`，改为 `padding: 0; background: transparent;`
- `.semester-date-tag`：去除 `background`、`border`、`border-radius`，改为仅字号颜色
- `.semester-arrow`：保持
- `.semester-edit-btn`：缩小为更低调的样式

#### 学期栏响应式修改（第805-807行）

- `.semester-bar-card`：适配新设计

#### 图例放大（第416-421行）

- `.as-legend-days`：`clamp(10px,1vw,13px)`（上限 12→13）
- `.as-legend-days-total`：`clamp(10px,1vw,13px)`（上限 12→13）
- `.as-legend-color`：`clamp(10px,1.2vw,15px)`（上限 14→15）

#### 响应式图例放大（第808-810行）

- 次按钮（个人课程表/班级课程表/当前月/全学期）：统一的胶囊内联样式
- 取消按钮：从 `style="width:100%;"` 改为完整胶囊样式

#### 导出统计放大（第4867-4878行）

- 容器：`gap:14px; padding:18px 22px;`
- 教师项：`font-size:14px; padding:7px 12px;`
- 总计项：`font-size:14px; padding:7px 12px;`

### 性能考虑

- `backdrop-filter: blur()` 在老旧设备上可能有性能影响，但由于仅应用于弹窗（非高频渲染区域），影响可忽略
- 所有修改均为纯 CSS/字符串替换，无运行时开销

### 日志与兼容性

- 不涉及数据逻辑修改，不影响数据存储或导出
- 字号放大确保在移动端（`@media(max-width:600px)`）也同步生效
- 磨砂玻璃在不支持 `backdrop-filter` 的浏览器会降级为普通半透明背景
